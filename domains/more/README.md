# 自定义域文件目录

本目录用于存放你基于历史文档生成的专属域文件。

## 使用方式

1. **上传历史文档**：将你的历史 PRD、产品规范、技术文档、竞品分析、合规清单等文件放入本目录。
2. **让 AI 生成域文件**：告诉 skill "基于 `domains/more/` 下的文档生成我的产品线域文件"。
3. **AI 读取并生成**：skill 会读取你上传的文档，提取产品线背景、技术约束、合规要求、竞品列表、写作习惯等信息，按 `references/domain-template.md` 的结构生成一份专属域文件。
4. **自动匹配使用**：生成的域文件保存在本目录，后续撰写 PRD 时，当检测到匹配你产品线的关键词，自动加载你的专属域文件。

## 支持的输入材料

| 材料类型 | 提取内容 |
|-|-|
| 历史 PRD | 章节结构习惯、术语、字段命名、竞品列表、合规要求、**写作风格** |
| 产品规范/白皮书 | 产品线定位、目标市场、平台基线、技术架构 |
| 竞品分析报告 | 默认竞品集合、对比维度 |
| 合规/隐私文档 | 合规检查清单、隐私要求、权限策略 |
| 技术架构文档 | 依赖项、平台约束、版本兼容基线 |
| 评审记录/会议纪要 | 评审关注点、常见问题、团队偏好 |
| 设计规范/组件库 | 交互术语、写作约束、UI 状态定义 |

## 写作风格自动提取

生成域文件时，skill 会从历史文档中自动提取你的写作风格偏好，写入域文件的「写作风格偏好」节。提取内容包括：

- **术语偏好**：你习惯用的产品术语和表达方式（如用「toast 提示」还是「轻提示」）
- **句式结构**：段落长度、条件逻辑写法、5.3 组织方式
- **文案标注**：UI 文案用什么符号包裹（「」/【】/ 下划线等）
- **禁用与偏好词汇**：你从不使用的表达 + 你高频使用的表达
- **章节详略**：哪些章节偏好详写、哪些偏好精简
- **输出格式**：Markdown / DOCX / 流程图偏好

这些风格偏好会在起草 PRD 时自动覆盖 `references/writing-style.md` 中的通用规则，让输出更贴合你团队的文档习惯。

### 风格学习示例

```
用户：我上传了 5 份历史 PRD，帮我生成域文件。

AI：我从你的历史文档中提取到以下写作风格：
- 术语：你用「弹窗」不用「对话框」，用「置灰」不用「禁用态」
- 句式：偏好短句 + 编号列表，单段不超过 3 行
- 文案标注：UI 文案用「」包裹
- 5.3 风格：具体表现子块用粗体小标题分隔，表格内不嵌套表格
- 详略：背景章节精简（3-5 句），5.3 详写（L3 为主）
- 禁用词：除通用去 AI 味清单外，你的文档从不使用"赋能""抓手""链路"

已写入域文件「写作风格偏好」节，后续起草会自动应用。
```

## 生成规则

- 生成的域文件遵循 `references/domain-template.md` 的完整结构。
- 文件名建议使用产品线缩写：如 `iot-smart-home.md`、`saas-crm.md`。
- 每个域文件顶部会包含触发关键词列表，供 SKILL.md Step 1 自动检测使用。
- 信息不足的章节标 `待补充`，后续可追加材料再更新。

## 示例对话

```
用户：我上传了几份历史 PRD 到 domains/more/，帮我生成一份我们 SaaS CRM 产品线的域文件。

AI：我读取了你上传的 3 份文档，提取到以下信息：
- 产品线：XX SaaS CRM
- 目标市场：国内企业客户
- 平台：Web + 移动端
- 默认竞品：Salesforce、飞书 CRM、纷享销客
- 合规重点：等保三级、个保法
- 写作习惯：偏好表格式 5.3、段落简短

已生成域文件 `domains/more/saas-crm.md`，后续提到 CRM、客户管理、销售线索等关键词时会自动加载。
```

## 注意事项

- 上传的原始文档仅用于提取产品线上下文，不会被写入最终 PRD。
- 生成的域文件可随时手动编辑调整。
- 多次上传新材料后可要求"更新我的域文件"，AI 会增量合并新信息。
- 不同产品线建议生成独立域文件，避免上下文冲突。
