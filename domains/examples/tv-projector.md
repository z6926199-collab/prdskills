## 平台与硬件默认假设

- 操作系统：Android TV（AOSP-based），版本跨度通常 Android 10-14
- 芯片平台：MTK / Amlogic / Realtek（按机型系列区分）
- 遥控器：IR + BT 双模为主，部分高端支持 NFC 配对 + 语音
- 显示：4K 为主流，部分 8K；HDR10/HLG/Dolby Vision 按机型区分
- 音频输出：内置扬声器 / HDMI ARC / eARC / 蓝牙音箱
- 网络：WiFi（2.4G/5G/6）+ 有线以太网，部分投影仪仅 WiFi
- 投影仪额外：自动对焦/梯形校正、短焦/超短焦、亮度流明段

## TV 领域技术知识

### 显示与画质

#### 分辨率与 HDR

| 规格 | 说明 | Spec 影响 |
|------|------|-----------|
| 4K UHD (3840x2160) | 当前主流 | 界面元素需适配 4K 渲染 |
| 8K (7680x4320) | 高端机型 | 需确认功能在 8K 机型上的表现 |
| HDR10 | 基础 HDR，静态元数据 | 画质设置类功能需注明 HDR 模式下的行为 |
| HDR10+ | 动态元数据，Samsung 主推 | 同上 |
| Dolby Vision | 动态元数据，需授权 | 涉及播放类功能时注明 DV 兼容性 |
| HLG | 广播 HDR 标准 | 直播/频道类功能需支持 |

#### 画质模式

TV 通常有多种画质模式（标准/电影/游戏/运动/自定义），功能在不同画质模式下的行为可能不同。涉及画面叠加层（OSD/弹窗/悬浮窗）时需确认是否受画质模式影响。

### HDMI 与信号源

#### HDMI 规格

| 版本 | 带宽 | 关键能力 | Spec 影响 |
|------|------|----------|-----------|
| HDMI 2.0 | 18Gbps | 4K@60Hz, HDR10 | 中低端机型，部分功能可能受限 |
| HDMI 2.1 | 48Gbps | 4K@120Hz, VRR, ALLM, eARC | 游戏相关功能需确认 HDMI 版本要求 |

#### CEC (Consumer Electronics Control)

- CEC 允许通过 HDMI 控制其他设备（一键开关机、音量联动）。
- 涉及多设备联动的功能需注明 CEC 支持情况。
- 边界场景：CEC 设备冲突、CEC 被用户关闭、非标准 CEC 实现的兼容。

#### 信号源切换

- TV 端切换信号源（HDMI1/2/3、ATV、DTV、AV）会触发 UI 层级变化。
- 功能如果在某个信号源下运行，需说明切换信号源时的行为（暂停/退出/保持后台）。

### 遥控器与输入

#### 遥控器类型

| 类型 | 特征 | 边界场景 |
|------|------|----------|
| IR（红外） | 无配对、需对准、无语音 | 无法反向通信，无电量上报 |
| BT（蓝牙） | 需配对、无需对准、支持语音 | 断连恢复、重新配对流程 |
| BT+IR 双模 | 主流方案 | IR fallback 时语音不可用 |
| NFC 配对遥控 | 高端款 | 首次配对流程差异 |

#### 语音输入

- **近场语音**：遥控器语音键，按住说话。
- **远场语音**：机器内置麦克风，唤醒词触发。投影仪通常无远场。
- Spec 中需注明语音触发的功能是否两种模式都支持。

#### 焦点交互

TV 端不支持触摸（极少数投影仪有触摸板遥控），所有交互通过焦点系统：
- 方向键移动焦点
- 确认键执行动作
- 返回键回退/关闭

### 电源与启动

#### 电源状态

| 状态 | 说明 | Spec 影响 |
|------|------|-----------|
| 冷启动 | 完全断电后开机 | 启动耗时长，功能初始化时序 |
| 快速启动（待机） | 低功耗待机，按遥控器即亮 | 功能恢复状态、网络重连 |
| 休眠/屏保 | 无操作一段时间后进入 | 功能是否需要在休眠前保存状态 |
| OTA 重启 | 升级后自动重启 | 功能数据是否跨重启保留 |

#### 开机行为

涉及开机体验的功能需注明：
- 是否在开机引导中出现
- 开机后是否自动启动/恢复上次状态
- 冷启动 vs 快速启动下的行为差异

### 网络

- WiFi 为主，部分支持有线以太网。
- 投影仪多数仅 WiFi，无以太网口。
- 网络状态变化频繁（WiFi 断连/切换/弱信号），所有联网功能需考虑：
  - 无网络时的降级策略
  - 弱网络时的超时处理
  - 网络恢复后的自动重试逻辑

### 音频输出

| 通路 | 说明 | Spec 影响 |
|------|------|-----------|
| 内置扬声器 | 默认输出 | 基线能力 |
| HDMI ARC | 单向音频回传至 Soundbar | 音频设置类功能需覆盖 |
| HDMI eARC | 支持无损音频（Atmos） | 高端机型 |
| 蓝牙音箱/耳机 | 配对连接 | 延迟问题、断连恢复 |
| S/PDIF 光纤 | 老式音响连接 | 低优先级兼容 |

涉及音频的功能（语音助手、提示音、媒体播放）需说明在各音频通路下的行为。

### OTA 升级

#### 升级策略

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 静默升级 | 用户无感知，后台下载+待机时安装 | 小版本补丁 |
| 提醒升级 | 通知用户，用户选择时机 | 常规版本 |
| 强制升级 | 不可跳过，倒计时后自动执行 | 安全漏洞修复 |

#### 升级相关考虑

- 新功能上线时老版本用户的过渡方案
- 升级后功能的默认状态（开/关/跟随之前设置）
- 升级失败/中断后的回滚行为
- 数据迁移：老版本数据结构到新版本的兼容

### 投影仪额外约束

| 维度 | 约束 | Spec 影响 |
|------|------|-----------|
| 亮度 | 流明段不同（500-3000 ANSI） | 界面配色需考虑低亮度环境可见性 |
| 对焦/校正 | 自动对焦 + 梯形校正 | 开机流程中可能插入校正步骤 |
| 投射距离 | 短焦/超短焦/长焦 | 交互区域大小不同 |
| 散热 | 持续运行发热 | 长时间运行的功能需考虑温控降频 |
| 便携性 | 部分产品支持电池供电 | 电量管理、低电量提醒 |
| 无内置调谐器 | 不接收电视信号 | 频道/EPG 相关功能不适用 |

## 合规要求

### 通用合规底线

- 个人数据采集必须明示同意，不可默认勾选。
- 权限申请遵循最小必要原则，不提前索要未使用的权限。
- 用户数据的存储位置和保留期限需在隐私政策中说明。
- 退出/注销账号后数据应在约定期限内删除。

### 内容安全

- 涉及推荐/展示内容的功能需考虑内容审核机制。
- 不同区域的内容分级标准不同（如儿童模式的年龄分级）。
- 涉及 UGC（用户生成内容）的功能需要举报/屏蔽机制。

### DRM 与内容保护

- 涉及付费内容播放的功能需支持 DRM（Widevine/PlayReady）。
- DRM 级别影响输出分辨率（L1 支持高清、L3 仅标清）。
- Spec 中注明对 DRM 级别的要求和降级策略。

## 5.3 功能说明写作约束（TV/投影仪）

- **禁止使用"点击"**：TV 端所有交互通过焦点系统，必须使用"选中并确认"、"按确认键"、"按返回键"等遥控器交互术语。
- 焦点流向必须完整描述：初始焦点位置、方向键移动规则、循环/边界行为。
- 涉及弹窗/对话框时，写明默认焦点在哪个按钮上。
- 涉及列表/网格时，写明滚动行为和焦点记忆逻辑。
- 信号源切换时的行为（暂停/退出/保持后台）必须说明。
- 开机/待机/休眠场景下的功能状态必须覆盖。
- 音频通路切换时的功能行为必须说明。
- 遥控器类型差异（IR/BT/语音）对功能的影响必须覆盖。

## 评审角色

Spec 评审涉及的角色及其关注点：

| 角色 | 关注重点 |
|------|----------|

| 合规/法务 | 隐私数据采集、权限使用、市场合规风险 |

## TV 专项评审检查项

除通用评审检查外，TV/投影仪产品需额外关注：

- [ ] 焦点交互：所有页面/弹窗的初始焦点、移动规则、边界行为是否完整
- [ ] 遥控器兼容：IR/BT/语音遥控器下功能是否均可使用
- [ ] 电源状态：冷启动/快速启动/休眠/OTA 重启下的行为是否覆盖
- [ ] 信号源：功能在不同信号源下的行为是否说明
- [ ] 音频通路：涉及音频的功能是否覆盖所有输出通路
- [ ] 网络降级：无网络/弱网络下的降级策略是否定义
- [ ] 画质模式：叠加层是否受画质模式影响
- [ ] CEC 边界：设备冲突、CEC 关闭、非标准实现的兼容
- [ ] 投影仪差异：亮度/散热/无调谐器等投影仪特有约束
- [ ] OTA 兼容：老版本用户过渡、数据迁移、回滚方案
