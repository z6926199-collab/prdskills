# 移动/平板领域上下文

本文件在 PRD 起草或修改流程开始前注入，作为产品/合规/技术判断的默认上下文。除非用户当次明确说明不同情况，否则按本文件的设定进行。

## 公司与产品线

- 公司：TCL，全球化经营。
- 产品线：TCL 通讯，海外手机 / 平板。
- 本领域主要服务平板 / 手机产品线。

## 平台基线

- 操作系统：基于最新主力 Android AOSP 版本开发。
- 谷歌服务：携带 GMS（Google Mobile Services）。
- 芯片平台：MTK 与高通混用。
- 内存 / 存储档位：按 SKU 定位差异化，PRD 中不预设具体档位。

## 目标市场

- 销售市场：北美、欧盟、英国、东南亚。
- **合规审查重点**：以 **欧盟（GDPR）** 与 **北美（CCPA / COPPA）** 为审查基准，原因是这两个地区合规要求更严格；其他地区合规以这两套为基线兜底。

### 合规检查清单（移动/平板）

- [ ] 个人数据采集明示同意，不默认勾选
- [ ] 权限申请遵循最小必要原则，不提前索要未使用的权限
- [ ] 用户数据存储位置和保留期限在隐私政策中说明
- [ ] 退出/注销账号后数据在约定期限内删除
- [ ] GDPR：数据主体访问权、删除权、可携带权、DPO 联系方式
- [ ] CCPA：不要出售我的个人信息入口、消费者知情权
- [ ] COPPA：涉及儿童用户时的年龄门控与家长同意流程
- [ ] 跨境数据传输：数据出境评估、标准合同条款（SCC）
- [ ] 跟踪/广告标识符：opt-in 机制、广告偏好设置

## 语言

- 产品支持 47 种语言。
- **PRD 输出范围以中英文为准**，多语言细节在 PRD 中不展开，统一标注「按既有多语言流程支持」或「待补充」。
- 兜底规则：若未匹配到系统语言，兜底使用英语。

## 性能基线

- 因芯片与内存档位按 SKU 浮动，性能审查默认按 **「中低端档位」** 为基线（TCL 手机的主要市场定位）。
- 中低端机型的性能风险作为风险审查的必查项。

## 5.3 功能说明写作约束（移动/平板）

- 交互描述默认使用触屏操作术语：点击、长按、滑动、拖拽、捏合。
- 涉及物理按键时注明：电源键、音量键、侧边指纹。
- 手势交互需描述完整轨迹：从屏幕边缘左滑返回、下拉通知栏等。
- 多窗口/分屏场景需说明窗口尺寸下限和交互焦点切换逻辑。
- 折叠屏场景（如适用）需注明展开/折叠态的行为差异。
- 不预设具体屏幕尺寸和分辨率，按适配规范标注「按既有多分辨率适配方案支持」。
- 默认按中低端设备性能约束：动画时长、内存占用、后台保活策略需保守设定。

## 风险审查补充（移动/平板）

除通用风险维度外，必须额外扫描：

| 风险维度 | 说明 |
|----------|------|
| 中低端性能 | 动画掉帧、内存不足、启动慢、后台被杀 |
| AOSP 兼容性 | 不同 Android 版本的行为差异、API 废弃 |
| GMS 认证 | 功能是否依赖 GMS，无 GMS 时的降级方案 |
| 跨境数据传输 | 数据出境合规评估、本地化存储需求 |
| 多语言适配 | RTL 布局、文字截断、日期/数字格式 |
| 设备碎片化 | 屏幕尺寸/分辨率/芯片差异导致的兼容问题 |
| 电池与发热 | 后台活动、传感器持续使用的功耗影响 |
| OTA 升级 | 跨版本数据迁移、功能默认状态变化 |

## 输出格式

- 默认输出本地 Markdown 文件。
- 暂不考虑飞书 / Confluence / 其他在线文档输出。后续若公司内部协作工具提供 API，再扩展输出层。

## 在 PRD 中的体现方式

- 领域背景信息**不需要单独成节**写进 PRD；它作为撰写时的默认上下文，影响以下章节的写法：
  - `2.1 背景说明`：描述产品/业务背景时默认 TCL 海外平板/手机语境。
  - `2.4 竞品分析`：默认对照上述竞品集合。
  - `六、非功能需求`：默认按欧盟 GDPR + 北美 CCPA/COPPA 进行合规审查。
  - 风险审查：默认按中低端机型性能、AOSP 兼容、跨境数据传输等维度扫描。
- 如果用户当次需求与本文件设定不同（例如只面向某一地区、只针对高端机型），以用户最新指令为准，并在 PRD 顶部 `callout` 或 `待确认事项` 中标注偏离点。
