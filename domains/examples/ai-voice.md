# AI 语音领域上下文

本文件为国内 AI 语音需求的 PRD 提供领域检查、历史背景和写作规范。目标是让 PRD 能支持体验评审、研发实现、测试验收、数据监控、隐私合规和上线决策。

## 产品线定位

- **产品线**：Walleve / 小T语音，TV 端 AI 语音产品线。
- **目标市场**：中国大陆市场。
- 历史能力围绕语音入口、半屏/全屏对话流、智能体、技能域、中控协议、端云协同、语音诊断和运营配置展开。
- PRD 中不要把它写成通用手机语音助手。默认检查电视端特有因素。

## 产品定位输入

生成 PRD 时收集：

- **目标产品线**：默认优先按 Walleve/小T语音电视端产品线理解；如用户明确说明，也可扩展到手机、平板、智屏、可穿戴、IoT 设备、车载、应用、系统助手或跨设备场景。
- **目标市场**：默认面向中国大陆市场；如涉及区域、品牌或渠道差异，单独说明。
- **用户分层**：普通用户、老年用户、儿童/家庭用户、无障碍用户、办公用户、驾驶用户、内容创作者、服务/运营用户。
- **核心价值**：解放双手、设备控制、内容搜索、任务完成、陪伴、知识问答、多模态助手、无障碍、售后/服务提效。
- **体验承诺**：更快、更简单、更自然、更准确、更安全、更私密、更本地化。

### Walleve 补充输入

- **机芯和版本**：例如 9653、TC8000、9655、语音 6.12、语音 6.15、Walleve 版本阈值。
- **端侧形态**：半屏对话流、全屏对话流、语音主页、智能体中心、系统设置、帮助与反馈、整机诊断、Launcher。
- **协作方**：终端、中控、云端、运营平台、语音底层、Launcher、帮助与反馈、系统设置、测试、数据。
- **配置和灰度**：中控配置、运营平台配置、设备范围、版本开关、灰度策略、回滚方式。

## 语音交互检查项

每个功能都要定义：

- **入口方式**：唤醒词、长按按键、电源键、耳机键、桌面图标、应用入口、小组件、控制中心、遥控器、车载/设备入口。
- **唤醒表现**：视觉状态、音效、震动、超时、误唤醒处理、多设备冲突处理。
- **用户说法**：标准话术和泛话术。
- **意图理解**：槽位提取、歧义处理、确认、纠错、取消。
- **对话模式**：单轮、多轮、追问、打断、上下文保持、会话结束。
- **响应方式**：TTS 语音回复、屏幕卡片、设备动作、应用跳转、通知、静默执行或混合反馈。
- **无障碍**：读屏、字幕/文本展示、震动反馈、大字体、低视力使用、听障用户使用。
- **电视遥控器交互**：焦点落点、上焦态、返回键、主页键、关机键、三色键、菜单键、设置键、音量键、按键是否打断当前语音流程。

## AI 能力依赖

判断需求是否依赖：

| 能力 | 检查内容 |
|------|----------|
| ASR | 语音识别、方言/口音处理、噪声鲁棒性、离线识别 |
| NLU/LLM | 意图分类、槽位填充、推理、内容生成、安全策略 |
| TTS | 音色、语速、情感、打断、多语言输出 |
| 唤醒词 | 关键词模型、误唤醒率、漏唤醒率、本地处理 |
| 声纹 | 注册、鉴权、家庭/多用户区分、隐私 |
| 设备控制 | 系统权限、应用意图、系统接口、IoT 协议、硬件能力 |
| 知识/服务 | 搜索、地图、天气、日历、联系人、媒体、客服、第三方接口 |
| 端云协同 | 本地兜底、云端时延、网络依赖、数据同步、模型版本 |

相关时覆盖以下场景：

| 风险类别 | 具体场景 |
|----------|----------|
| 网络 | 无网络、弱网、网络切换、需要登录网络 |
| 云端服务 | 云服务超时、模型不可用、接口失败、调用额度或频率限制 |

| 设备 | 设备离线、硬件能力缺失、低电量、低存储、高温、处理器/内存压力 |
| 交互 | 连续语音输入、长时间会话、重复唤醒、快速连续指令 |
| 多用户 | 多用户/家庭使用、儿童保护、声纹不匹配、锁屏限制 |
| 跨设备 | 多台设备同时唤醒或执行同一指令 |
| 敏感内容 | 个人数据、金融/健康/法律建议、未成年人、政治/文化合规 |

### 历史能力主线

#### 1. 大模型引导语策略

- 大模型上线后，需要对大模型和非大模型场景提供统一的上下文引导语策略。
- 过去大模型与非大模型引导语策略不统一，数据源不同。
- 不支持大模型或未打开大模型开关的设备，沿用原引导语策略，由中控从运营平台获取引导语数据。
- 支持并开启大模型服务的设备，优先展示大模型生成引导语，并使用兜底引导语。

**关键规则**：

- 半屏对话流：基于本次对话前五轮内容生成引导语；缓存时间为 5 分钟；不足五轮按已有轮次生成；超过五轮取最近五轮。
- 全屏对话流：基于进入智能体全屏对话流后的最近五轮内容生成引导语。
- 半屏和全屏均固定展示三条引导语。
- 每条引导语至少 2 个字，最多 15 个字；支持中文和英文，不支持符号、阿拉伯数字等字符。
- 大模型生成内容含敏感词、字数/字符不符合要求时，用兜底引导语替代。
- 网络超时或数据返回超时时，固定展示网络诊断、语音诊断、帮助与反馈等可跳转引导。
- 免打扰、后台音乐、连续对话等状态下，第一个位置固定展示退出状态引导语。

#### 2. 智能体全屏对话流

- 年底大模型上线，目标是沉浸式交互和主动式服务，追赶小米、海信在大模型沉浸式交互和智能体内容矩阵上的表现。
- 覆盖 9653、TC8000、9655 机芯用户，语音 6.12 版本开始体验。

**关键能力**：

- 入口包括智能体中心、语音主页、语音话术、半屏对话流点击智能体卡片。
- 首次进入时展示智能体背景图、智能体形象、开篇招呼语、开篇引导语。
- 进入前需判断账号登录、大模型开关、儿童模式、智能体是否下架、云端数据是否获取成功。
- 对话中展示用户 query、生成内容卡片、AI 生成提示、TTS 播报跟随加粗、点赞/点踩、引导语。
- 退出方式包括语音退出、遥控器返回/主页/关机/三色键、系统重启等。
- 智能体技能白名单用于控制哪些系统指令在全屏对话流内执行，非白名单统一由大模型回复。

**历史风险**：

- 数据获取失败或超时需要进入报错页，并明确 Toast 文案。
- 智能体下架需要中断进入并返回前页。
- 生成中被打断、网络异常、生成失败、点赞点踩是否出现都需要定义。
- 儿童模式下可能不支持相关能力，需要 TTS 提示。
- 点赞点踩二级分类不宜过多，点踩分类可非必填，避免用户负担。

#### 3. 语音诊断

- 原语音诊断偏基础和建议型，无法真正帮助用户分析和处理语音问题，对减少客诉作用有限。
- 新方案迭代为一站式诊断与修复，增加重启、清除语音数据、一键修复、升级语音等策略。
- 用户范围为语音版本不低于 6.15 的机型用户。

**核心流程**：

- 入口包括语音主页、整机诊断/帮助与反馈、系统设置、语音指令。
- 进入前需校验隐私协议和用户体验改进计划。
- 诊断首页包括功能介绍区、信息展示区、诊断策略区。
- 先诊断后修复：远场语音诊断、近场/遥控器语音诊断。
- 不诊断直接修复：一键修复，清除语音应用缓存并重启电视。

**关键风险**：

- 一键修复会清除应用缓存并重启电视，无法保留原设置项和已同意隐私协议信息，需要二次确认和风险告知。
- 一键修复过程不可被打断。
- 升级语音可能杀掉原进程，用户需重新进入语音诊断。
- 内存不足时引导到电视卫视清理加速页面。
- 重复诊断失败超过次数后，应展示客服联系方案。

#### 4. 一句话多意图

- 竞品和车/手机多端教育了用户，一句话表达多个指令逐渐变成普遍能力。
- 抽样数据显示，多意图表达虽然占比不高，但按生产话术规模估算仍有六位数量级。

**关键规则**：

- 支持拆解用户话术、多意图执行、意图依赖判断、执行顺序重排、白名单、冲突判断、重复判断、TTS 处理。
- 拆解出的多个意图必须在白名单内；不在白名单内则过滤。
- 剩余 0 个可执行意图时，不拆解，整体走原意图理解。
- 按序执行意图；能判断成功/失败的意图，达到成功定义后再执行下一个；不能判断时按执行时间等待。
- 当前 TTS 策略为接受指令后、执行前播报"好的"；长期方案可用 agent 总结执行结果。
- 组合指令执行过程中，远场/近场/免唤醒词/三色键/主页键触发可打断，未执行意图不再执行。
- 智能体中心不支持一句话多指令。

#### 5. Launcher 与 Voice 协作

**职责边界**：

- Voice：意图识别、参数解析、技能路由、失败兜底话术。
- Launcher：页面承接、参数校验、焦点落点、返回键处理、失败 UI。

**历史风险**：

- Voice 与 Launcher 使用不同字段含义会导致参数语义不一致。
- 修改协议前必须由双方 Owner 确认，并补充联调用例。

## 数据与验收指标

优先使用可量化指标：

| 指标 | 说明 |
|------|------|
| 唤醒成功率 | 唤醒成功次数 / 唤醒尝试次数 |
| 误唤醒率 | 非主动唤醒次数 / 单位时间 |
| ASR 识别成功率 | 正确识别次数 / 识别尝试次数 |
| 意图理解成功率 | 正确理解次数 / 意图识别次数 |
| 任务完成率 | 完成任务次数 / 开始任务次数 |
| 平均响应时延 | 从用户说完到系统响应的平均时间 |
| 多轮对话完成率 | 完成多轮对话的会话 / 开始多轮的会话 |
| 兜底/不支持意图占比 | 兜底次数 / 意图识别总次数 |
| 权限授权转化率 | 授权通过数 / 权限弹窗展示数 |
| 语音功能日活/周活/月活 | 使用语音功能的独立用户数 |
| 人均语音交互次数 | 语音交互总次数 / 语音活跃用户数 |
| 首次使用语音后的留存 | 首次使用后 N 日仍使用的用户比例 |
| 语音场景 NPS/满意度 | 语音场景专项满意度评分 |
| 按问题类型拆分的投诉/工单率 | 各类问题的投诉量 / 语音活跃用户数 |

每个指标都要说明定义、分母、统计周期、事件来源和验收方式。

## PRD 必查问题

每次生成 PRD 都要检查：

- 是否属于新功能、历史功能迭代、策略统一、问题修复、体验优化或数据治理。
- 是否影响终端、中控、云端、运营平台、语音底层、Launcher、帮助与反馈、系统设置、整机诊断等多方。
- 是否有版本门槛、机芯范围、灰度配置、回滚方案。
- 是否涉及远场/近场、麦克风、遥控器语音键、唤醒词、免唤醒词。
- 是否涉及半屏/全屏对话流、焦点、卡片、Toast、TTS、遥控器按键。
- 是否涉及大模型开关、账号登录、儿童模式、智能体下架、云端数据超时。
- 是否需要兜底：运营平台兜底、固定引导语兜底、网络诊断、语音诊断、帮助与反馈、客服。
- 是否需要运营平台配置：仅当涉及运营内容、话术/引导语、服务卡片、资源推荐、配置项开关、场景集或高频调整策略时才平台化；固定逻辑和低频变更不要强行平台化。
- 是否涉及隐私协议、用户体验改进计划、麦克风、悬浮窗、语音数据。
- 是否需要新增或复用测试集：中控标准集、泛化集、终端存量用例、必过集。
- 是否需要读取线上数据：机芯语音数据、域分布、PV/UV、点击率、诊断结果、queryid。

## PRD 生成检查清单

- [ ] 区分大模型/非大模型、开关状态、设备支持状态
- [ ] 明确引导语来源：大模型、运营平台、中控返回、终端固定
- [ ] 写清兜底策略、敏感词策略、超时策略、无网络策略、状态引导语策略
- [ ] 写清引导语是否支持点击和语音说
- [ ] 每个入口都写前置条件、数据获取、页面展示、失败兜底
- [ ] 半屏与全屏卡片宽度、焦点、样式可能不同，不要默认一致
- [ ] 遥控器焦点、按键响应和返回路径必须写清
- [ ] 数据统计需区分入口来源、智能体、半屏/全屏、点赞/点踩、引导语点击/语音触发
- [ ] 诊断类需求写入口、版本限制、隐私校验、能力开关校验、诊断流程、修复按钮优先级、错误码分组、不可打断操作
- [ ] 写清白名单、冲突列表、依赖关系、重复去重、槽位级差异和执行顺序
- [ ] 涉及 Launcher 协作时补充双方职责、协议字段、版本兼容、灰度配置、回滚方式和失败兜底
