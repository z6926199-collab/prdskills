本文件为 HDMI 相关功能的 PRD 提供技术背景和写作规范。

- 公司：TCL
- 产品线：TV/投影仪 HDMI 软件
- 负责模块：HDMI 设备控制(CEC)、自诊断、连接引导
- 涉及机型：MT9655 / MT9653 / TC8000

## CEC 协议知识

### CEC 基础

- CEC（Consumer Electronics Control）是 HDMI 规范中的设备控制协议。
- 允许通过 HDMI 线缆在设备间传递控制指令（一键开关机、音量联动、信号源切换）。
- CEC 在 HDMI 物理层使用单线通信（Pin 13）。

### CEC 常用命令

| 命令 | Opcode | 说明 |
|-|-|-|
| Image View On | 0x04 | 电视开机 |
| Active Source | 0x82 | 声明自己为活动信号源 |
| Inactive Source | 0x9D | 声明自己非活动信号源 |
| Request Active Source | 0x85 | 请求当前活动信号源 |
| Set Stream Path | 0x86 | 切换到指定 HDMI 口 |
| Report Power Status | 0x90 | 报告电源状态 |
| User Control Pressed | 0x44 | 用户按键透传 |
| Set Menu Language | 0x32 | 设置菜单语言 |

### CEC 关键风险

- **误唤醒**：外接设备发送 Active Source / Image View On 导致电视被意外开机。
- **设备冲突**：多个 CEC 设备同时声明 Active Source。
- **CEC 被关闭**：用户在设置中关闭 CEC 后所有联动失效。
- **非标准实现**：部分设备 CEC 实现不完全符合规范。
- **断电上电**：设备重新上电时可能触发电视跟随开机。

## EDID 知识

- EDID（Extended Display Identification Data）是显示器向信号源报告自身能力的标准数据。
- 包含：支持的分辨率/刷新率、音频能力、色域信息。
- 涉及分辨率切换的功能需确认 EDID 读取是否正常。
- EDID 读取失败时需要降级策略。

## HDMI 信号状态机

| 状态 | 说明 | 转换条件 |
|-|-|-|
| 无信号 | HDMI 口无输入 | 设备断电/拔线 |
| 信号检测中 | 检测到插入，正在协商 | 物理连接建立 |
| 信号锁定 | 协商完成，画面正常 | EDID + HDCP 握手成功 |
| 信号丢失 | 显示中断 | 设备休眠/断电/线缆松动 |
| CEC 就绪 | CEC 通道建立 | CEC 握手完成 |

## 5.3 写作约束（HDMI 域）

- UI 元素名称用【】标注（如【电视跟随开机】、【取消】、【确定】）。
- 每个字段内容用编号分条。
- 英文翻译放在「相关定义说明」字段中。
- TV 端交互使用焦点术语，不用"点击"。
- 弹窗需写明默认焦点、倒计时规则、确认按钮可用条件。

## 合规检查

- CEC 控制涉及用户设备行为，需在设置中提供明确开关。
- 误唤醒风险需通过二次确认或用户提示缓解。
- 设备信息读取（EDID）不涉及个人数据，合规风险低。

## 风险审查补充

| 风险维度 | 说明 |
|-|-|
| CEC 兼容性 | 非标准设备导致命令不响应或误触发 |
| 信号稳定性 | HDMI 线缆质量/长度影响信号 |
| 多设备冲突 | 多台 CEC 设备同时操作 |
| 固件差异 | 不同机芯的 CEC 实现差异 |
