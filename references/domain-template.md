### 5.3 写法偏好

> 5.3 是 PRD 最核心的章节，团队在此往往有强个人习惯。从历史文档中重点提取以下维度：

- 默认深度档位：（团队常用 L2 / L3 / 按功能分别判断）
- 子块组织方式：（粗体小标题分块 / 编号列表 / 嵌套表格）
- 异常场景写法：（表格式 / 编号列表 / 条件句段落）
- 状态描述方式：（状态表格 / 状态枚举列表 / 引用流程图）
- 埋点写法：（内嵌在具体表现子块 / 单独附表 / 仅引用 6.1）
- 边界/逆向/异常/极限详略：（每列 3+ 条 / 仅覆盖高风险 / 按套餐 checklist）
- 功能拆分粒度偏好：（按场景切 / 按页面切 / 按能力切 / 按状态切）
- 单张 5.3 表的典型长度：（简洁型 ~500 字 / 标准型 ~1000 字 / 详尽型 ~2000 字）
