# PRD Writer Unified

> 融合 5 条产品线的 AI 辅助 PRD 写作 Skill，支持 TCL 手机/平板、TV/投影仪、HDMI 硬件、AI 语音/Walleve + 通用模式。

**触发方式**：用户输入 `/prd-writer` 或说「用 prd-writer skill」时激活。对话中提到 PRD 不会自动触发。

---

## 核心能力

- **全量撰写**：从需求背景到完整 PRD 一站式生成
- **早期探索**：梳理模糊想法，输出概念版 PRD 并冻结范围
- **局部修改**：对已有 PRD 做精确修改，不破坏原有结构
- **PRD 评审**：8 大类 Review + 5 大类风险自审，只出结论不改原文
- **证据追溯**：为原始素材分配编号，在 PRD 中标注来源
- **多产品线支持**：通过域文件注入不同产品线的上下文和规范
- **去 AI 味**：自动移除套话，替换为具体行为/状态/反馈

## 设计原则

- 严格 9 列功能说明表格，确保覆盖所有维度
- 不臆造内容，不确定处标「待确认」
- 域文件写作风格偏好优先于通用规则
- 质检自动化：去 AI 味 + 风险审查 + Review 三合一

## 使用模式

### 模式 1：全量撰写

**适用场景**：有明确的需求背景和功能列表，需要一份完整 PRD。

**你需要提供**：需求背景（为什么做）+ 功能列表（做什么）+ 可选：目标用户、竞品、技术约束

**你会得到**：完整 PRD 草稿 + 待办事项清单 + 证据追溯表（如提供原始素材）

### 模式 2：早期想法探索

**适用场景**：只有一个模糊想法或 MVP 概念，还没想清楚范围。

**流程**：
1. AI 每轮最多问 3 个问题帮你厘清方向
2. 输出概念版 PRD（核心用户 + 痛点 + MVP 功能 ≤3 条 + 不做什么）
3. 范围冻结确认 → 进入全量撰写

### 模式 3：局部修改

**适用场景**：已有一份 PRD，需要修改/补充/调整某些部分。

**规则**：只改你要求的部分，其余原文不动；保持原 PRD 格式和术语风格。

### 模式 4：PRD 评审

**适用场景**：有一份 PRD 想做质量检查。

**你会得到**：8 大类 Review + 5 大类风险自审 + 按严重程度排序（🔴必须/🟡建议/🟢优化）

## 工作流程

```
Step 0  请求分流 ─────────→ 判断路径：全量 / 探索 / 修改 / 评审
  │
Step 1  产品线上下文注入 ──→ 检测关键词 → 加载域文件
  │
Step 2  项目类型判断 ──────→ 0-to-1（全新）or 1-to-100（迭代）
  │
Step 3  缺口澄清 + PRD 深度 → 3-6 题问卷 + 深度档位选择
  │
Step 4  证据收集（可选）──→ 分配证据编号，构建上下文模型
  │
Step 5  场景树 + 功能大纲 ─→ 拆解场景 → 输出大纲 → 用户确认
  │
Step 6  起草 PRD ──────────→ 9 列表格写 5.3，L1-L4 深度分层
  │
Step 7  质检 ──────────────→ 去 AI 味 + 风险自审 + Review
  │
Step 8  输出 + 迭代 ──────→ PRD 正文 + 待办清单；小改走局部修改，大改回 Step 5
```

## 5.3 功能说明表格规范

每个功能单元使用 **严格 9 行两列格式**：

| 字段 | 内容 |
|---|---|
| 功能 | [功能名称] |
| 功能详细说明 | [说明] |
| 功能入口/前置条件 | [入口和前置] |
| 具体表现 | [6 子块展开] |
| 相关定义说明 | [术语定义] |
| 边界场景 | [边界覆盖] |
| 逆向场景/退化逻辑 | [降级兜底] |
| 异常场景 | [异常处理] |
| 极限场景 | [极限覆盖] |

### 「具体表现」6 子块

1. **用户流程**：成功路径 + ≥2 条异常路径
2. **功能逻辑**：系统判断顺序、条件分支
3. **页面与状态**：各状态的进入/退出条件和可见表现
4. **字段与文案**：输入校验、各场景下的提示文案
5. **数据与埋点**：存储位置、同步策略、埋点事件
6. **关联事件**：上下游依赖、通知、回调

### 深度分层（L1-L4）

| 层级 | 适用范围 | 说明 |
|---|---|---|
| L1 极简 | P2 探索性功能、纯 UI 适配 | 一句话简述 |
| L2 标准 | P1 中低风险功能 | 用户流程 + 逻辑 + 异常≥3 类 |
| L3 完整 | P0 核心功能、MVP | 6 子块全部展开 |
| L4 深度 | 账号/支付/隐私/AI/硬件 | L3 + 协议/容错/回滚/安全 |

## 安装方式

### 方式一：直接安装（推荐）

对 AI 说：

```
安装 skills：[你的压缩包路径]
```

AI 会自动解压并安装到 `~/.claude/skills/prd-writer`。

### 方式二：手动安装

```bash
# 克隆仓库
git clone https://github.com/z6926199-collab/prd-writer-unified.git

# 创建符号链接
ln -s /path/to/prd-writer-unified ~/.claude/skills/prd-writer
```

安装完成后输入 `/prd-writer` 即可使用。

## 快速上手

### 第一次使用

```
你：/prd-writer 帮我写一个关于「用户积分商城」的 PRD

AI：
1. 确认产品线（或通用模式）
2. 出 3-5 道可点击问题补齐关键决策
3. 展示功能大纲请你确认
4. 生成完整 PRD + 待办清单
```

### 只做评审

```
你：/prd-writer [粘贴 PRD] 帮我 review

AI：输出 8 类 Review + 5 类风险结论，不改原文
```

### 小范围修改

```
你：/prd-writer [粘贴 PRD] 把 5.3.2 的异常场景补充完整

AI：只改目标章节 + 列出联动待办 + 局部风险检查
```
